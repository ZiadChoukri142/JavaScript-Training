- forEach exécute une fonction fournie une fois pour chaque élément du tableau.

- map contient des paires clé-valeur et se souvient de l'ordre d'insertion d'origine des clés. Toute valeur (à la fois des objets et des valeurs primitives) peut être utilisée comme clé ou comme valeur.

- filter crée un tableau rempli de tous les éléments du tableau qui passent un test

- reduce exécute une fonction de réduction (que vous fournissez) sur chaque élément du tableau, ce qui donne une valeur de sortie unique.

- every teste si tous les éléments du tableau passent le test implémenté par la fonction fournie. Il renvoie une valeur booléenne.

- some teste si au moins un élément du tableau passe le test implémenté par la fonction fournie. Il renvoie une valeur booléenne.

- find renvoie la valeur du premier élément du tableau fourni qui satisfait la fonction de test fournie. Si aucune valeur ne satisfait la fonction de test, undefined est renvoyé.

- findIndex renvoie l'index du premier élément du tableau qui satisfait la fonction de test fournie. Sinon, il renvoie -1, indiquant qu'aucun élément n'a réussi le test.

- intersection nous donnera les éléments que les deux tableaux ont en commun, dans ce cas, le résultat doit être [1,5].

- union doit être la plus simple de toutes, à la fin, le résultat devrait être tous les éléments de A, tous de B,

- with était destinée à fournir un raccourci pour écrire des accès récurrents aux objets
